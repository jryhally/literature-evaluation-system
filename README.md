# 文学作品评价体系

一个可部署到 GitHub Pages 的文学作品多维评价与选书导航系统原型。

这个项目的出发点不是“给文学作品排座次”，而是帮助阅读经验不多、但想养成阅读习惯的人，在面对大量经典作品时快速形成概念：

- 这本书强在哪里？
- 它适合我现在读吗？
- 阅读难点在哪里？
- 应该带着什么问题去读？

输入文学作品名称和作者后，系统会尝试从公开百科资料中获取作品介绍和作者背景，并输出作品评分、阅读推荐度、作品优势和阅读方向。

## 功能

- 输入作品名和/或作者，二者至少填一个
- 先展示相似作品候选，用户点击确认后再评分
- 点击确认作品后，系统会优先根据作品资料推断真实作者，并展示对应作者介绍
- 左侧展示：
  - 文学作品介绍
  - 作者背景
- 右侧展示：
  - 总分
  - 阅读推荐度
  - 多维评分
  - 评分依据
- 下方展示：
  - 作品优势
  - 阅读方向
  - 适合人群
  - 阅读提醒
- 纯前端实现，可直接部署到 GitHub Pages
- 无需数据库、无需后端

## 评价维度

本项目参考文学奖项常见评审关注点，设计为 7 个维度：

| 维度 | 权重 | 含义 |
| --- | ---: | --- |
| 文学艺术性 | 18% | 语言风格、象征系统、审美密度与文本完成度 |
| 思想与人文深度 | 18% | 人的处境、伦理困境、社会历史经验与精神问题 |
| 原创性与创新 | 14% | 题材、形式、视角和叙事技术上的开创性 |
| 结构与叙事完成度 | 14% | 人物、情节、节奏、结构组织和整体控制力 |
| 文化影响力 | 16% | 传播、经典化、获奖、改编和文学史位置 |
| 普遍性与跨文化性 | 10% | 是否呈现可被广泛理解的人类经验 |
| 阅读难度 | 10% | 语言、结构、思想密度造成的阅读挑战 |

> 阅读难度不是作品好坏的直接判断，它表示文本可能需要的阅读耐心、背景知识和解释成本。

## 面向新读者的阅读推荐度

除了文学总分，项目还会计算一个“阅读推荐度”。它更贴近“我现在该不该读这本书”的问题。

阅读推荐度会综合考虑：

- 作品影响力
- 叙事完成度
- 思想深度
- 文学艺术性
- 阅读难度
- 是否有入门友好信号，例如短篇、散文、现实主义、成长主题等
- 是否有高门槛信号，例如意识流、现代主义、哲学性、实验性、长篇史诗等

因此，一本作品文学总分很高，但阅读推荐度不一定最高。比如某些现代主义经典非常重要，却不一定适合作为重启阅读习惯的第一本。

## 评分方式

当前版本是透明启发式模型，不是 AI 模型，也不是权威文学评奖。

系统会先搜索相似作品候选，避免在书名不完整、译名不同、或书名和作者不匹配时直接误判。

用户点击确认具体作品后，系统会读取作品介绍、作者背景、分类信息，并根据以下信号生成分数：

- 文学关键词
- 思想/历史/社会关键词
- 奖项信号，例如诺贝尔文学奖、茅盾文学奖、布克奖、普利策奖等
- 经典化与传播信号，例如经典、代表作、翻译、改编、世界文学等
- 公开资料丰富程度

## 数据来源

当前版本使用中文维基百科公开 API 检索摘要。

如果需要更严肃的版本，可以继续接入：

- Open Library
- Wikidata
- 豆瓣/Goodreads 等第三方数据源
- 自建文学作品数据库
- 大模型 API，用于阅读摘要、评论和学术资料后的综合判断

## 本地运行

进入项目目录：

```bash
cd LiteratureEvaluationSystem
```

启动一个本地静态服务器：

```bash
python3 -m http.server 4180
```

打开：

```text
http://127.0.0.1:4180
```

## 部署到 GitHub Pages

1. 在 GitHub 新建仓库，例如 `literature-evaluation-system`
2. 上传本项目全部文件
3. 进入仓库 `Settings`
4. 打开 `Pages`
5. Source 选择 `Deploy from a branch`
6. Branch 选择 `main`，目录选择 `/root`
7. 保存后等待 GitHub Pages 构建完成

## 免责声明

文学评价本身具有强烈的历史、文化和审美主观性。本项目输出的分数只能作为阅读、研究和讨论的辅助参考，不应视为权威结论。
